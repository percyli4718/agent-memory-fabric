# v0.1 Milestone

## Goal

Ship a minimal but credible open source core that proves cross-session memory search through MCP with a
small, safe, and inspectable feature set.

## Deliverables

### 1. Core model

- define `memory_record` schema
- define scope, visibility, and type enums
- define retention and redaction states

### 2. Storage

- SQLite reference backend for local development
- PostgreSQL reference backend for team deployments

### 3. MCP server

- implement `search_memory`
- implement `write_memory`
- implement `get_recent_project_context`
- implement `get_decisions`
- implement `get_open_questions`

### 4. Policy enforcement

- require scope and visibility on writes
- require author and source metadata on writes
- validate all tool inputs against schema

### 5. Developer experience

- local quickstart example
- sample `AGENTS.md`
- sample Codex and Claude Code integration docs

### 6. Project quality

- `README.md`
- `SECURITY.md`
- initial CI workflow placeholders
- issue templates and contribution notes

## Exit Criteria

- a local user can run the MCP server
- a coding agent can write a structured memory record
- a new session can query and retrieve that memory
- the retrieval is bounded by repository or project scope
- the basic docs explain architecture and limitations clearly

## Nice-to-Haves

- optional embedding provider interface
- basic ranking strategy abstraction
- simple audit log output
